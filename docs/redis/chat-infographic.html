설계 포인트 ① — Pub/Sub 클라이언트 분리
왜 두 개의 Redis 연결을 만드는가?
subscriber.subscribe()를 호출한 연결은 구독 전용 상태가 되어 publish() 같은 일반 명령을 실행할 수 없습니다.
// redisPubSub.ts
export const publisher ← 발행 전용
export const subscriber ← 구독 전용
두 연결 모두 같은 Redis 서버에 연결되지만
역할을 분리하여 충돌 방지

  • 이력 조회 가능
설계 포인트 ③ — 동적 채널 구독
서버 시작 시가 아니라 입장 시 구독
채팅방에 첫 사용자가 입장하는 순간 해당 채널을 구독합니다. 사용 중인 채널만 구독하므로 불필요한 Redis 트래픽이 없습니다.
// websocket.ts
wss.on('connection', (ws, req) => {
  rooms.set(roomId, new Set())
  subscriber.subscribe(`chat:${'{roomId}'}`)
})
rooms Map: 연결된 WebSocket 관리
subscriber: Redis 채널 실시간 수신
