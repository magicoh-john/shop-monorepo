# Redis 개념

## 왜 필요한가

지금까지 사용한 저장소들은 두 가지 한계 중 하나를 가지고 있습니다.

> **localStorage** — 브라우저에만 존재. 서버에서 읽을 수 없음  
> **PostgreSQL** — 서버에서 읽을 수 있지만, 디스크 I/O로 인해 속도가 느림

자주 읽고 쓰이지만 영구 보존이 필요하지 않은 데이터(장바구니, 세션, 채팅 메시지 중계)를  
빠르게 처리할 수 있는 **서버 측 저장소**가 필요합니다.

---

## 무엇인가

Redis는 데이터를 **메모리(RAM)에 저장하는 서버**입니다.

디스크에 저장하는 PostgreSQL과 달리 RAM에 저장하므로 읽기/쓰기 속도가 극단적으로 빠릅니다.  
또한 만료 시간(TTL)을 설정하면 일정 시간 후 데이터가 자동으로 삭제됩니다.

| | PostgreSQL | Redis |
|---|---|---|
| 저장 위치 | 디스크 | 메모리(RAM) |
| 속도 | 보통 | 매우 빠름 |
| 데이터 유지 | 영구 | 설정한 TTL까지 |
| 적합한 데이터 | 주문, 회원, 상품 | 장바구니, 세션, 채팅 |

---

## 어떻게 쓰는가

### 기본 key-value 저장

```
SET cart:user_123  '[{"productId":"abc","name":"티셔츠","price":29000}]'
GET cart:user_123
DEL cart:user_123
```

### 만료 시간(TTL) 설정

```
SET cart:session_abc  '[...]'  EX 604800    ← 7일 후 자동 삭제
```

### Pub/Sub — 메시지 중계

여러 서버 간에 메시지를 주고받을 수 있는 기능입니다.  
한 서버가 메시지를 발행(publish)하면, 구독(subscribe) 중인 모든 서버가 수신합니다.

```
# 서버 1: 메시지 발행
PUBLISH chat:room_1  '{"from":"user_A","text":"안녕하세요"}'

# 서버 2: 구독 중이면 자동 수신
SUBSCRIBE chat:room_1
```

---

## 이 프로젝트에서의 역할

| 기능 | 키 구조 | TTL |
|---|---|---|
| 비로그인 장바구니 | `cart:{sessionId}` | 7일 |
| 채팅 메시지 중계 | `chat:{roomId}` (Pub/Sub) | — |
